1. Kursidentität

1. Course identity

Grunddaten

Core data

Arbeitstitel: Nachlass, Erbschaft & Orientierung im Trauerfall

Working title: Inheritance, estate & guidance in times of loss

Kursziel: Laien in einer emotional und praktisch belastenden Situation ruhig, verständlich und Schritt für Schritt orientieren.

Course goal: Guide non-experts calmly, clearly, and step by step through an emotionally and practically difficult situation.

Zielgruppe: Angehörige, Vorsorgende, Familien, Partner, Kinder, Menschen ohne juristische Vorkenntnisse.

Target group: Relatives, people preparing ahead, families, partners, children, and users without legal background.

Kernhaltung

Core tone

  • ruhig statt hektisch
  • laienfreundlich statt juristisch trocken
  • praktisch statt abstrakt
  • entlastend statt überfordernd
  • mit klarer Reihenfolge statt Chaos
  • calm instead of hectic
  • accessible instead of legalistic
  • practical instead of abstract
  • relieving instead of overwhelming
  • clear order instead of chaos
Dieser Kurs soll Menschen nicht nur informieren, sondern sie durch eine schwierige Lage tragen.
This course should not only inform people, but help carry them through a difficult situation.

2. Verbindliche Kursregeln

2. Binding course rules

Design

Design

  • einheitliches CSS-System für den ganzen Kurs
  • helle, ruhige Oberfläche mit Türkis-Akzenten
  • gleiche Karten-, Hero- und Panel-Logik
  • keine Fremdstile ohne Entscheidung
  • one shared CSS system across the course
  • light calm interface with turquoise accents
  • same card, hero, and panel logic
  • no external style drift without decision

Technik

Technical

  • DE/EN mit fester Sprachlogik
  • nur lokale Pfade
  • nur Inline-SVGs
  • Local Storage für Notizen und Arbeitsbereiche
  • Druckfunktion für nutzerrelevante Inhalte
  • DE/EN with fixed language logic
  • local paths only
  • inline SVG only
  • local storage for notes and work areas
  • print support for user-relevant content

Arbeitsregel

Work rule

  • erst Masterdatei pflegen, dann neue Seite bauen
  • jede neue Seite bekommt festen Pfad und Status
  • keine improvisierten Dateinamen
  • keine ungeplanten Abzweigungen
  • update master page first, then build new pages
  • every new page gets a fixed path and status
  • no improvised file names
  • no unplanned detours
